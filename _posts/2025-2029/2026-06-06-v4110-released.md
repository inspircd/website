InspIRCd v4.11.0 is now available, [download it from the releases page](https://github.com/inspircd/inspircd/releases/tag/v4.11.0).

The most notable changes in this release are:

- Added extended ban `B:` (bot) to the botmode module.

- Added support for forwarding TLS fingerprints from HAProxy using a custom TLV.

- Fixed sending malformed line-wrapped FJOINs for large channels in the v3 protocol compatibility layer.

- [Fixed the ldapauth and ldapoper not escaping user-provided values that are used in a search filter.](https://docs.inspircd.org/security/2026-01)

- Fixed the log_json module not writing messages correctly when built against yyjson.

<!--more-->

No breaking configuration changes have been made since the v4.0.0 release. Please consult [the list of breaking changes](https://docs.inspircd.org/4/breaking-changes) if you are upgrading from v3.

The full change log can be found [on the docs site](https://docs.inspircd.org/4/change-log#inspircd-4100) or [on GitHub](https://github.com/inspircd/inspircd/compare/v4.10.1...v4.11.0).
